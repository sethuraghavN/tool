Change Management is the addition, modification or removal of anything that could have an effect on IT Services

The Scope should include all IT Services, Configuration Items, Processes, Documentation etc.

Release management is the A collection of hardware/software documentation, Processes or other Components required to implement one or more approved Changes to IT Services.

Service Requests are usually handled by a Service Desk, and do not require an RFC to be submitted.

Note that a parent ticket can contain any of the child classifications below it. This means that a Release can contain any Problems, for example, without a Change between them in the hierarchy.

When a parent ticket, such as a Release, is updated, a technician can elect to update/close all the child tickets, such as Changes, Problems, Major Incidents, and Incidents/Service Requests. that are nestled within it.
